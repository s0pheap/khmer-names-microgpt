# MicroGPT — Khmer Name Generator

A tiny GPT language model built from scratch in pure Python (no PyTorch, no TensorFlow) that generates Cambodian names in Latin script. Based on [Karpathy's microgpt.py](https://gist.github.com/karpathy/8627fe009c40f57531cb18360106ce95), trained on real Khmer name data, served via FastAPI with a browser UI.

## How It Works

```
khmer_names.csv  ─┐
                  ├─► build_dataset.py ─► khmer_names_training.txt ─► train.py ─► .json
extra_names.csv  ─┘                                                        │
                                                                           ▼
                                                              convert_checkpoint.py ─► .npz
                                                                           │
                                                               api.py ─► /generate ─► frontend
```

1. `khmer_name_pool.py` — generates Khmer name combinations, converts to Latin, saves `extra_names.csv`
2. `build_dataset.py` — reads both CSVs, expands to 3 styles (given / surname / full), deduplicates, writes `khmer_names_training.txt`
3. `train.py` — trains a character-level GPT, saves `checkpoint_khmer_names.json`
4. `convert_checkpoint.py` — converts the JSON checkpoint to compressed NumPy `.npz` (5× smaller)
5. `api.py` — loads `.npz` at startup, serves `/generate`
6. `frontend/index.html` — single-page UI served by FastAPI

## Setup

**1. Install dependencies**

```bash
pip install -r requirements.txt
pip install khmer_latin_name_transformer
```

**2. Generate extra names** *(optional — skip if `data/extra_names.csv` already exists)*

```bash
python utils/khmer_name_pool.py
```

**3. Build training data** *(skip if `data/khmer_names_training.txt` already exists)*

```bash
python utils/build_dataset.py
```

**4. Train the model** *(skip if checkpoints already exist)*

```bash
python train.py
```

**5. Convert checkpoint to NumPy**

```bash
python utils/convert_checkpoint.py
```

**6. Start the server**

```bash
uvicorn api:app --reload
```

Open [http://localhost:8000](http://localhost:8000) in your browser.

## Training Data Format

Each name pair is expanded into 3 styles before writing:

```
# Input: ("សេង វិជ្ជា", "seng vichhea")
#
# Expands to:
វិជ្ជា           ← given name
vichhea
សេង              ← surname
seng
សេង វិជ្ជា       ← full name
seng vichhea
```

Duplicate pairs are removed before writing.

## Checkpoint Formats

| Format | File | Size | Used by |
|--------|------|------|---------|
| JSON   | `checkpoint_khmer_names.json` | ~120 KB | human-readable, inspectable |
| NumPy  | `checkpoint_khmer_names.npz`  | ~25 KB  | `api.py` (5× smaller, faster to load) |

`train.py` saves JSON. Run `convert_checkpoint.py` to produce the `.npz` for the API.

## Model

| Key           | Training Data             | Description                                          |
|---------------|---------------------------|------------------------------------------------------|
| `khmer_names` | `khmer_names_training.txt` | Character-level GPT on Khmer + Latin name pairs |

Hyperparameters: `n_layer=1`, `n_embd=16`, `n_head=4`, `block_size=16`, `steps=3000`, optimizer: Adam with linear LR decay.

## API

**`POST /generate`**

```json
{
  "model": "khmer_names",
  "prefix": "dara",
  "count": 10,
  "temperature": 0.8
}
```

Response:

```json
{
  "results": ["dara chea", "dara sok", "..."],
  "prefix": "dara",
  "model": "khmer_names"
}
```

**`GET /models`** — list loaded models  
**`GET /health`** — health check  
**`GET /`** — serves the browser UI

## Architecture

`microgpt.py` is Karpathy's original self-contained script (untouched). `train.py` and `api.py` implement the same decoder-only transformer independently:

- Token + positional embeddings
- Multi-head self-attention (scaled dot-product)
- 2-layer MLP with ReLU
- RMSNorm before each sub-layer
- Residual connections
- Custom autograd `Value` class — no ML framework needed
